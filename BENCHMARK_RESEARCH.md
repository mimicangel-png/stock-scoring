## 二、每个项目怎么解决"评分→买卖"问题

### 2.1 Qlib — 评分→组合→信号 三层转换

```
因子计算(Alpha158/360)
    ↓
ML模型预测(XGBoost/LSTM/Transformer)  ← 评分不是手工加权，是模型学出来的
    ↓
截面组合优化(MOSEK QP solver)       ← 把预测分数转为多空权重
    ↓
执行器生成订单                       ← 权重变化 → 具体买卖指令
```

**关键设计**：
- **Point-in-Time 数据库**：每个时间戳只暴露该时点已知的数据，彻底杜绝前瞻偏差
- **Rolling Retraining**：模型定期重训以适应市场变化
- **ExpressionCache**：因子计算只算一次，大幅加速

**怎么用到我们项目上**：
- 因子输出改为连续值 → GLM 学习权重 → 截面排序 → 做多 Top-N
- 这已经比当前"手工加权 + 固定阈值"好一个层级

### 2.2 QUANTAXIS — 统一的"时间戳=担保"

```
QIFI 协议: (code, price, date, quantity)  ← 买卖指令带上精确时间
    ↓
QAData 内存数据库: 严格按时间推进        ← 不会拿到"未来"的数据
    ↓
QARS Rust 引擎: 高性能执行              ← 10x 提速
```

**关键设计**：
- **回测 API = 实盘 API**：`account.buy("000001", 10.5, "2025-01-15", 1000)` 在回测和实盘中的行为一致
- **零拷贝数据桥**：Pandas ↔ Polars ↔ Arrow 无拷贝转换

**怎么用到我们项目上**：
- 交易模拟 API 设计成 `(code, price, date, shares)` 格式
- 回测和实盘用同一套接口，避免"回测能跑、实盘报错"

### 2.3 1984tkr/multi-factor-stock-selection — 因子自我淘汰

```
因子计算 → IC(信息系数)评估 → ICIR(IC稳定性)跟踪
    ↓
ICIR < 0.3 连续3个月 → 自动移入 factor_graveyard/
    ↓
存活因子 → 加权评分 → Top N 选股 → 均线/成交量择时 → 回测
```

**关键设计**：
- **因子墓地机制**：不稳定的因子自动失效，而不是无限堆叠
- **择时层独立于选股层**：选股告诉你买什么，择时告诉你现在能不能买
- **完整绩效指标**：Sharpe/Calmar/Sortino/IR/盈亏比/换手率，不只是 IC

**怎么用到我们项目上**：
- 给 SS 的每个加减分因子加 IC 跟踪，连续 IC 走弱就降权或移除
- 分离"选股层"和"择时层"，避免把大盘风险混入选股信号
- 绩效指标从"因子 IC"升级到"组合级指标"

### 2.4 Walk-Forward Backtester — 时间的诚实性

```
时间轴: [train₁: D1-D252] [test₁: D253-D378]
              [train₂: D127-D378] [test₂: D379-D504]
                    [train₃: D253-D504] [test₃: D505-D630]
```

**关键设计**：
- **固定窗口滚动**：train_size=252天, test_size=126天, step=126天
- **参数稳定性检验**：如果最优参数在不同窗口间波动很大，说明在过拟合
- **评估只看 test 窗口**：所有 Sharpe/收益只统计 out-of-sample 部分

**怎么用到我们项目上**：
- 直接参考它的窗口切分逻辑，替换目前的"单次回测"
- 参数稳定性检验：如果 SS 的最佳阈值不同窗口差 10+ 分，说明阈值不可靠

### 2.5 arxiv 2507.07107 — 学术级最佳实践

**最值得借鉴的 5 个设计**：

#### (1) Purge Cross-Validation（清洗交叉验证）

```
训练集 [D1 —— D100]   ← 训练用
         purge [D101-D105]   ← 5天间隔，防止T+10标签泄露
测试集         [D106 —— D120]  ← 测试用
```

当前我们的 GLM 用 T+10 收益做训练目标，但训练集和测试集之间没有 purge gap。如果一只股票在 training set 的最后一天买入，T+10 收益正好落在 test set。这就是标签泄露。

#### (2) 截面相对排序（Cross-Sectional Ranking）

不做绝对收益预测（"这只股票会涨 3% 还是 5%"），而是做截面排序（"这 137 只里，哪些排名前 20%"）。

因为：绝对收益预测需要预测市场方向，而截面排序只需要判断相对优劣——噪音小得多。

#### (3) 自适应中性化

根据市场波动状态动态调整中性化强度：
- 高波动期：加强中性化（市场乱，信号不可靠）
- 低波动期：放松中性化（信号更有价值）

#### (4) 组合优化中的交易成本建模

不是简单假设"买入按收盘价"，而是从三个维度建模：
- 买卖价差（spread cost）
- 市场冲击（market impact）
- 佣金（commission）

#### (5) 市场状态分段评估

分别在牛市/熊市/震荡市中评估策略表现。如果策略 80% 的收益来自 2024 年 9 月的暴涨行情，那这个策略不可靠。

## 四、我们项目可以直接引入的设计

### 短期可做（1-2天，不改核心架构）

1. **因子墓地机制** — 给每个 SS 加减分因子加 IC 追踪列，连续 IC 走弱自动降权
2. **Purge Gap** — 训练/测试之间插 5 天空白，消除 T+10 标签泄露
3. **绩效指标升级** — 加 Calmar/Sortino/盈亏比/换手率/ICIR
4. **参数稳定性检验** — 不同 WFO 窗口的最优阈值是否一致

### 中期改造（从架构层面）

1. **评分转为截面排序** — 不关心绝对分数，只关心"全市场 137 只中排名第几"
2. **组合级回测** — 用 `(code, price, date, shares)` API 模拟真实交易
3. **分离选股和择时** — 选股层给排名，择时层判断"今天能不能买"
4. **市场状态分段评估** — 牛市/熊市/震荡各算一遍 Sharpe

### 长期（如果需要更完整的系统）

1. Qlib 风格的 PoT 数据库
2. 自适应中性化（借鉴论文的 β_vol 参数）
3. 多模型集成（XGBoost + GLM，而非纯 GLM）

---

## 五、一句话总结

**业界做"评分到买卖"的最佳模式是：因子连续化 → ML 截面排序 → 组合优化 → 时间戳保真的交易模拟 → 多窗口 Walk-Forward 验证。**

我们的 SS 评分系统在这一链条的前两步（评分连续化、截面排序）有最大的改进空间。
