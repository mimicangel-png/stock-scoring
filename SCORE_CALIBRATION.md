# 评分为什么需要"标尺"：从因子 IC 到可交易买卖点

> 2026-07-03 补充分析

---

## 一、当前回测究竟在测什么

### 因子 IC 分析（当前做法）

```
评分(T日) → 与 T+N 日收益率的相关性(IC) → "IC=0.18，不错"
```

**IC 高了代表什么？** 评分高的股票，10天后平均涨得比评分低的多。

**IC 不等于什么？**
- 不等于"按评分买能赚钱"（你不知道买点在哪）
- 不等于"评分 75 分就是买入信号"（可能已经涨了5天才到75分）
- 不等于"评分降到 45 分就该卖"（可能是在洗盘）

### 举个例子，说明 IC 的欺骗性

| 场景 | 评分 | T+10 收益 | IC 表现 |
|------|:---:|:--------:|:-----:|
| 股票 A | 85 分 | +8% | IC 贡献大 |
| 股票 B | 85 分 | -2% | IC 贡献小 |

IC 高只是因为 A 类样本多于 B 类。但没有人告诉你：
- **A 是靠什么涨的？** 如果 A 的 85 分是因为"连续涨停后评分飙高"，那你在 85 分买入正好接盘
- **B 为什么没涨？** 如果 B 的 85 分来自基本面改善但市场没反应，那 85 分可能是真正的买入机会

**同一个 85 分，A 和 B 的买卖逻辑完全不同。**

## 二、从"评分准不准"到"评分能不能用"

评分系统的终极验证只有一条：

> 如果我在过去两年里，每天只看评分做交易，严格执行买卖规则，最终年化收益是多少？

这需要回答四个问题：

### 问题 1：在哪个时点买入？

| 策略 | 示例 | 问题 |
|------|------|------|
| 评分首次突破阈值 | SS 从 55→75，触发买入 | 如果突破是因为当天涨停呢？ |
| 评分 + 回踩确认 | SS 75 分且股价回调到 MA20 | 可能等不到回踩就飞了 |
| 评分 + 趋势确认 | SS 75 分且连续3天上升 | 可能已经涨完了 |
| GLM 领先 SS 信号 | GLM 70→85 但 SS 还在 55 | GLM 看对了但 SS 没跟上的时候 |

**没有一个通用答案。不同评分模式对应不同的买入逻辑。**

### 问题 2：什么时候卖出？

| 条件 | 示例 | 问题 |
|------|------|------|
| 评分跌破阈值 | SS 从 75→55 | 可能只是技术性回调 |
| 连续评分下降 | SS 连续降 3 天 | 滞后止损 |
| 固定持有期 | T+5 / T+10 强制卖出 | 但有些股票 T+3 就走坏了 |
| 止盈止损 | +15% 止盈 / -8% 止损 | 阈值怎么定？回到"手工过拟合" |

### 问题 3：买多少？

| 策略 | 问题 |
|------|------|
| 评分高的多买 | 可能重仓了一只假信号 |
| 等权重 | 浪费了评分的区分度 |
| Kelly 公式 | 需要知道真实的胜率和赔率 |

### 问题 4：不同评分段的买卖逻辑一样吗？

**答案是不一样。**

- 评分 80+ 的股票：动量策略 — 追趋势，止损紧
- 评分 60-75 的股票：反转策略 — 等回踩，止损宽
- 评分跌到 40- 的股票：如果已经在持仓中，应该基于什么逻辑卖出？

**同一个评分系统，对高分区和低分区的买卖规则应该不同。**

## 三、评分分数应该被"标定"成什么

核心问题：**ss_score=65 意味着什么？**

目前 65 分 = 基分 50 + 技术加分 + 资金加分 + 信息加分 — 扣分。这是一个**相对排名**，不是**绝对概率**。

需要的标定：

```
ss_score=65 → 历史上在这个分数区间买入，持 T+5 天的胜率是 X%，平均收益是 Y%
ss_score=80 → 历史上在这个分数区间买入，持 T+5 天的胜率是 Z%，平均收益是 W%
ss_score=40 → 如果持有中，历史上有 P% 概率接下来会继续下跌 >5%
```

**这就是"标尺"：每个分数值对应一个可依赖的统计量。**

标定的来源：
- 从历史回测中统计每个评分区间的**真实交易结果**（不是因子 IC）
- 按评分分段 → 假设在评分触发时买入 → 按预设规则卖出 → 统计胜率/赔率/最大回撤
- 这些统计量就是评分分数的"定义"

### 举例：理想中的分数标尺

| SS 分 | 历史胜率(T+10) | 平均收益 | 最大回撤 | 建议策略 |
|:---:|:---:|:---:|:---:|------|
| 85+ | 72% | +4.2% | -3.1% | 趋势跟踪，止损-4% |
| 70-84 | 58% | +2.1% | -5.8% | 回调买入，止损-6% |
| 55-69 | 47% | +0.3% | -8.2% | 持有不动/不加仓 |
| 40-54 | 38% | -1.8% | -12.5% | 只卖不买 |
| <40 | 22% | -5.1% | -18.4% | 立即卖出 |

有了这个标尺，**分数才有实际意义**。一个 72 分和一个 68 分的差距不是 4 分的数字游戏，而是"历史上 72 分的胜率高出 10 个百分点"。

## 四、如何建立这个标尺

### 步骤 1：构建历史"交易模拟"数据集

不跑因子 IC。直接模拟：

```python
for each historical date D:
    scores = compute_scores(D)   # 只用 D 当天及之前的数据
    for each stock with score > threshold:
        buy_price = close[D]     # 假设以收盘价买入
        for each exit_rule:
            exit_date = find_exit(D, buy_price, exit_rule)
            trade_return = (close[exit_date] / buy_price - 1)
            record: {score, entry_date, exit_date, return, exit_reason}
```

每条记录是一次真实的交易模拟，不是因子相关性。

### 步骤 2：按评分分段统计

把历史上所有模拟交易按评分分桶（75-79、70-74、65-69…），每个桶统计：
- 胜率（>0 的交易占比）
- 平均收益
- 收益分布（是少数大赚拉高均值，还是普遍小额正收益？）
- 最大连续亏损次数
- 不同退出规则下的表现差异

**关键：不仅要看均值，还要看分布的尾部。** 如果 80 分桶的平均收益是 +4%，但 80% 的收益集中在 5% 的股票上，那这个 80 分实际上是一个"彩票信号"。

### 步骤 3：标定每个分数段的"最佳策略"

不同分数段的历史表现会告诉你：
- 高分段的标的适合**趋势跟随**（因为动量持续）
- 中等分数段的标的适合**均值回归**（因为信号不稳定但估值合理）
- 低分段的标的**不能碰**（因为大概率继续跌）

### 步骤 4：用标定后的策略跑真正的 Walk-Forward

这轮才是最终验证：
- 训练期：用前 60% 时间的数据标定分数→策略映射
- 测试期：用后 40% 的数据按标定好的策略交易
- 评估：年化收益、Sharpe、最大回撤、胜率

如果测试期的表现和训练期标定的统计量一致，说明分数是有意义的、策略是可复现的。

## 五、跟现有代码的对接

现有代码中，`backtest_final.py` 和 `backtest_glm_comprehensive.py` 已经在做 Walk-Forward GLM 训练。问题在于：

1. **GLM 的目标是 T+10 收益**，但训练完成后没有生成交易信号，只是把 GLM 分加到报告里
2. **SS 的评分阈值是手工设的**（≥75=强烈买入），不是从历史标定来的
3. **没有完整的买卖点模拟**

改进方案：在现有 Walk-Forward 框架中加入**交易模拟层**：

```
现有：特征 → GLM 预测 → 评分 → 跟 T+10 收益比 IC
改进：特征 → GLM 预测 → 评分 → 触发买卖信号 → 模拟交易 → 统计真实胜率/收益
```

具体就是在 `backtest_glm_comprehensive.py` 或新脚本中，把：
```python
# 现有
Y10.append((kl[idx+10]['close']/c0-1)*100)  # 只看因子 IC

# 改为
signal = generate_trade_signal(ss_score, glm_score, score_history, price_history)
simulated_trades.append(execute_trade(signal, price_data))
```

然后再从 `simulated_trades` 统计真实的交易指标。

## 六、一句话

**评分不应该只是一个数字，而应该是"在这个分数上按照某个规则买卖，历史上证明能赚钱"的承诺。** 建立这个承诺的唯一方式，就是把评分映射到完全模拟的交易行为上，用真实的交易结果来验证和标定分数。
